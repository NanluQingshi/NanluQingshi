# 前端校招生转全栈 & AI时代成长计划

> 写给迷茫的自己：AI时代不是让程序员消失，而是让"能用AI放大自己"的人价值翻倍。
> 你现在有前端基础，转全栈是最自然的路径，后续再看是否往AI方向延伸。

---

## 一、整体思路

```
当前状态         → 已在职，前端基础有，TS/框架/工程化不够扎实
阶段零（当下-1个月）→ 前端薄弱点补强，同时启动全栈，不要等"准备好了再开始"
阶段一（0-6个月）  → 扎稳全栈基础，能独立交付完整功能
阶段二（6-12个月） → 工程能力提升，能负责一个模块的全链路
阶段三（12-24个月）→ 按兴趣/市场分叉：AI应用方向 或 算法/大模型方向
```

**核心原则**
- 以"能交付"为驱动，不以"看完课"为驱动
- AI工具（Cursor、Claude Code、Copilot）从第一天就用，不要等"学完再用"
- 每个阶段做一个完整项目，比刷100道题更值钱

**关于"前端没学扎实就去学全栈"的焦虑**

> 不需要先"补牢前端"再"学全栈"。两件事可以并行，而且全栈项目本身会逼着你把前端补扎实。
> 框架用法、工程化、TypeScript 这三块，**带着真实问题去学**比单独刷知识点效率高 10 倍。
> 唯一值得专门花时间的是 TypeScript——因为全栈前后端都要用，TS 的问题会持续拖慢你。

## 六、AI工具使用习惯（贯穿始终）

> 这不是一个阶段，而是一种工作方式，从第一天就要建立。

**日常工具**
- **Claude Code / Cursor**：写代码时实时问、让 AI 帮你 review、解释不懂的代码
- **Claude.ai**：学习新概念时对话式理解，比看文档快
- **GitHub Copilot**：IDE 内代码补全

**用AI学习的正确姿势**
- 遇到不懂的概念：先让 AI 用类比解释，再看文档
- 写完代码：让 AI 做 code review，找潜在问题
- 卡 bug：把错误信息和上下文给 AI，通常10秒解决
- 不要让 AI 帮你"全写"，要让 AI 帮你"更快理解和迭代"

## 七、执行建议

### 时间分配（工作日）
```
每天 1.5-2 小时学习
├── 30min 看理论/文档
├── 60min 写代码/做项目
└── 15min 复盘/记录
```

### 每周检查
- 本周完成了哪些 TODO？
- 卡在哪里了？是知识问题还是执行问题？
- 下周优先级是什么？

### 避免的坑
1. **不要同时学太多**：一个时间段只跟一个主线，其他的"了解即可"
2. **不要只看不做**：看10小时视频不如做1个项目
3. **不要追新技术**：把一个技术学扎实，比10个技术都懂一点强
4. **不要跳过部署**：项目不上线等于没做，部署过程会暴露大量问题
